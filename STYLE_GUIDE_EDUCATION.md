# Education Style Guide

## Tone
- Professional, clear, and instructional
- Original synthesis (no textbook copying/paraphrase)
- Authoritative but readable; plain language before technical depth
- If external references are used, cite only public/official sources actually consulted and restate concepts in fresh wording

## Section template (required per chapter)
1. Introduction
2. Why this matters
3. Learning outcomes
4. Core concepts
   - Plain-language explanation
   - Technical explanation
   - Application layer
   - Risk/compliance/real-world caveat
5. Scenario set (minimum 3 applied scenarios per chapter)
6. In practice card
7. Mini case study
8. Common misunderstandings
9. Practical checklist
10. Chapter roundup
11. Knowledge-check questions
12. Model answers with reasoning and weaker-alternative explanation

## Question-bank baseline
- Minimum 4 exam-style knowledge-check questions per chapter
- Each question includes:
  - correct answer key
  - reasoning for correct option
  - explicit explanation of weaker alternatives

## Callout taxonomy
- **Exam Tip**: exam strategy and answer framing guidance (expandable)
- **Pitfall**: common misunderstanding or frequent error pattern
- **Practitioner Note / In Practice**: practical workflow or client-facing execution example
- **Compliance Alert**: governance, conduct, legal, or documentation risk warning

## Consistency rules
- Use shared glossary definitions across Unit 1 and Unit 2
- Keep terminology stable (especially suitability, appropriateness, best execution, conflicts)
- Maintain chapter ordering logic from foundations to application
- Keep learning framing explicit: educational guidance, not legal/tax/investment advice
